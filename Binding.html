2. Property Binding

Attribute vs Property

1. Attributes and property are not the same.
2. Attributes are defined by html, properties are defined by Document Object Model(DOM)...typescript class in this case.
3. Attributes initialize DOM properties and then they are done. Attribute values cannot change once they are initialized.
4. Property values however can change.

8. Component Interaction

AppComponent is the parent for all other components in angular. Test is an example of child component. Parent & child components interact using @Input() & @Output() decorator.


Parent to child data transfer: @Input()

Steps:

1. Create a variable/property in app.component.ts in the component class (like ParentVariableSendingData in our case)
2. In app.component.html, inside the opening tag of the test-component tag/selector(app-test in our case), bind a child-component variable/property to the app.component property just creeated. ([ChildVariableReceivingParentData]="ParentVariableSendingData" in our case)
3. In test.component.ts, add Input into the first line of import modules.
4. Create a child-component variable as defined in the app.component.html.(ChildVariableReceivingParentData in our case).This variable/property will receive the parent data. Can also create an alias for the property as shown in this file near variable definition.
5. Add @Input() just before public keyword in the child-component variable declaration.

Child to parent data transfer(through events): @Output()

Steps:

1. Child-component sends data to parent-component using events.
2. Import 'EventEmitter' and 'Output' class on the first line of import modules list.
3. Create a new instance of the EventEmitter class.(ChildEvent in our case)
4. To send this ChildEvent to app.component(parent), add @Output() just before the just declared ChildEvent instance.
5. Now we need to trigger the event on some user action. Here we use a button click as the trigger. Add a new button tag in the test.component.ts template with event binding on type click event and a method defined in the component class of the test.component.ts.
6. In the fireEvent method, call the emit() method with the ChildEvent instance of the EventEmitter, with the message/to-be-sent-data inside the paranthesis. This will send the data to the app.component.html.
7. To catch the data at app.component.html, inside the opening tag of test-component tag/selector, create a new variable that will catch the data(answer in our case), equate it to $event which contains the event data and bind this to the event ChildEvent of the test.component.ts.
